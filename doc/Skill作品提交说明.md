# PPT Agent Skill — 专业演示文稿全流程 AI 生成助手

## 一句话价值点

**用对话替代设计：一句话需求输入，AI 自动完成万元/页级别 PPT 的完整工作流，从调研到可编辑 PPTX 输出，端到端提效 95%+。**

---

## Skill 创新价值说明

传统 PPT 制作存在「设计门槛高、内容填充靠编、风格不一致、耗时长」四大痛点。市面现有 AI PPT 工具（如 Gamma、美图 AI PPT）多采用「大纲→套模板」的浅层方案，产出千篇一律、信息密度低、无法在 PowerPoint 中编辑。

**PPT Agent Skill 的核心创新**：不做模板套用，而是**模拟万元/页专业 PPT 设计公司的完整工作流**，将其编码为 AI Agent 可执行的 6 步 Pipeline：

| 创新点 | 说明 |
|--------|------|
| **策划与设计分离** | 先验证信息结构（策划稿 JSON），再做视觉包装（HTML 设计稿），避免"边做边改"的低效循环 |
| **内容驱动版式** | 不是选模板再填内容，而是由内容特征自动决定布局（Bento Grid 决策矩阵，7 种布局 × 6 种卡片类型） |
| **真实数据填充** | 通过搜索工具获取真实数据，杜绝 AI 凭空杜撰的"幻觉数据" |
| **全管线可编辑输出** | HTML → SVG → PPTX 全链路打通，PPT 365 中右键"转换为形状"即可编辑文字和形状，非图片贴图 |
| **跨平台 Skill 架构** | 纯 Markdown 定义的 Skill，可被 Claude Code、Cursor、Windsurf 等任意 AI Agent 平台加载执行 |

## Skill 核心亮点

### 1. 专业级 6 步 Pipeline

```
Step 1: 需求调研（7 题三层递进访谈）
  ↓
Step 2: 资料搜集（多工具并行，真实数据）
  ↓
Step 3: 大纲策划（金字塔原理，结构化 JSON）
  ↓
Step 4: 内容分配 + 策划稿（每页 ≥3 卡片，≥2 种类型）
  ↓
Step 5: 风格决策 + 智能配图 + 逐页 HTML 设计稿
  ↓
Step 6: 后处理（HTML → preview.html → SVG → PPTX）
```

### 2. 8 种预置风格一键切换
暗黑科技 / 小米橙 / 蓝白商务 / 朱红宫墙 / 清新自然 / 紫金奢华 / 极简灰白 / 活力彩虹，通过 CSS 变量系统实现全局风格一致性。

### 3. Bento Grid 智能布局
7 种精确布局规格（单一焦点 → 混合网格）+ 6 种卡片内容类型（text/data/list/process/tag_cloud/data_highlight）+ 决策矩阵自动选择最优布局。

### 4. HTML → SVG → PPTX 全链路转换
- `html_packager.py`：多页合并为可翻页预览
- `html2svg.py`：DOM 直接转 SVG，保留 `<text>` 可编辑（非截图）
- `svg2pptx.py`：OOXML 原生 SVG 嵌入，PPT 365 中可转为可编辑形状

### 5. 管线兼容性保障
专门编写的 `pipeline-compat.md` 文档，从 HTML 源头规避 CSS 禁止特性（伪元素、conic-gradient、mask-image 等），确保转换零丢失。

## Skill 使用步骤

### 前置条件
- 任意支持 Skill 加载的 AI Agent 平台（Claude Code / Cursor / Windsurf 等）
- Python 3.x（用于后处理脚本）
- Node.js（用于 HTML → SVG 转换，可选）

### 使用流程

```
1. 将 ppt-agent-skill 目录放在工作目录下
   （或通过平台的 Skill 加载机制引入）

2. 对话触发：
   "帮我把 doc/ 目录下的报告做成 PPT"
   "做一个关于 XX 的演示文稿，15 页，暗黑科技风"
   "我要给老板汇报 XX 项目进展"

3. AI Agent 自动执行 6 步 Pipeline：
   调研 → 搜集 → 大纲 → 策划 → 设计 → 后处理

4. 产物输出在 ppt-output/ 目录：
   - preview.html  → 浏览器打开即可预览
   - presentation.pptx → PowerPoint 打开即可编辑
   - slides/ → 独立 HTML 页面
   - svg/ → 矢量 SVG 文件
```

### 自动化运行配置

通过 `settings.json` 预授权权限，可实现全程零人工干预：

```json
{
  "permissions": {
    "allow": ["Bash(.*)", "Read", "Write", "Edit", "Glob", "Grep"]
  }
}
```

## Skill 开发相关

### 技术架构

```
ppt-agent-skill/
├── SKILL.md                 # Skill 主定义（6 步 Pipeline 指令集）
├── references/
│   ├── prompts.md           # 5 套可复用 Prompt 模板
│   ├── style-system.md      # 8 种预置风格 + CSS 变量系统
│   ├── bento-grid.md        # 7 种布局规格 + 决策矩阵
│   ├── pipeline-compat.md   # HTML→SVG→PPTX 管线兼容性规则
│   └── method.md            # 核心方法论
└── scripts/
    ├── html_packager.py     # 多页 HTML 合并为翻页预览
    ├── html2svg.py          # HTML → SVG（Puppeteer + dom-to-svg）
    └── svg2pptx.py          # SVG → PPTX（OOXML 原生嵌入）
```

### 技术实现细节

#### 开发平台选择
- **Skill 定义层**：纯 Markdown + JSON Schema，平台无关，任何支持 system prompt 注入的 AI Agent 均可加载
- **后处理层**：Python + Node.js 混合技术栈
  - Python：python-pptx（OOXML 操作）、lxml（XML 解析）、Pillow（图片处理）
  - Node.js：Puppeteer（无头浏览器渲染）、dom-to-svg（DOM 到 SVG 矢量转换）、esbuild（模块打包）

#### 核心技术创新

**1. 结构化 Prompt 工程**
- 5 套 Prompt 模板形成完整链路，每步输出 JSON 作为下一步输入
- 策划稿 JSON 结构精确定义 page_type / layout_hint / cards[] / card_type，消除 LLM 输出的随机性
- 通过硬性约束（每页 ≥3 卡片、≥2 种 card_type、≥1 张 data 卡片）保证输出质量下限

**2. CSS 变量驱动的风格系统**
- 所有颜色通过 CSS 变量引用，禁止硬编码，一键切换全局风格
- 60-30-10 色彩比例法则 + accent 色使用约束，确保"高级感"而非"花哨"
- 排版系统定义 7 级字号阶梯（H0→Caption），严格的间距层级

**3. HTML → SVG → PPTX 管线**
- `html2svg.py`：基于 Puppeteer 在无头浏览器中渲染 HTML，通过 dom-to-svg 将 DOM 直接转为 SVG 矢量图，保留 `<text>` 元素可编辑性
- `svg2pptx.py`：将 SVG 通过 OOXML 原生 SVG 嵌入到 PPTX 中，PPT 365 中右键"转换为形状"即可编辑
- 兼容性规则文档 `pipeline-compat.md` 从 HTML 源头规避 15+ 种 CSS 禁止特性，确保转换零丢失

**4. 自适应降级策略**
- 环境感知：自动检测可用工具能力（搜索/图片生成/脚本执行），缺失时自动降级
- 复杂度自适应：根据页数自动调整调研深度、搜索量、分批策略
- 跨页视觉叙事：密度交替、章节色彩递进、封面结尾呼应、渐进揭示

### 创作思路

#### 从业务痛点到方案的推导

```
痛点分析：
  企业 PPT 制作 = 高频 + 高耗时 + 高门槛 + 低质量
    ↓
现有方案不足：
  模板工具 → 千篇一律，无法表达复杂业务逻辑
  AI PPT 工具 → 大纲→套模板的浅层方案，输出不可编辑
  外包设计 → 成本高（万元/页），沟通成本大，迭代慢
    ↓
核心洞察：
  专业 PPT 公司的价值不在"设计"，而在"工作流"：
  调研→素材→大纲→策划→设计→交付，每步有严格方法论
    ↓
方案设计：
  将这套工作流编码为 AI Agent 可执行的 Skill
  用结构化 JSON 在步骤间传递信息，消除 LLM 输出随机性
  用 CSS 变量系统保证全局风格一致性
  用 HTML→SVG→PPTX 管线实现可编辑输出
```

#### 核心模块设计逻辑

| 模块 | 设计逻辑 | 衔接方式 |
|------|---------|---------|
| 需求调研（Prompt #1） | 7 题三层递进，场景→内容→执行 | 输出需求 JSON |
| 大纲策划（Prompt #2） | 金字塔原理，结论先行 | 输入需求 JSON，输出大纲 JSON |
| 策划稿（Prompt #3） | 内容分配 + 布局决策一体化 | 输入大纲+素材，输出策划卡 JSON 数组 |
| 设计稿（Prompt #4） | CSS 变量 + Bento Grid + 管线约束 | 输入策划卡+风格 JSON，输出 HTML |
| 后处理（scripts/） | HTML→preview→SVG→PPTX | 输入 HTML 文件，输出 PPTX |

## Skill 推广建议

### 推广策略

1. **内部试点**：选择 3-5 个高频 PPT 制作场景（如月度汇报、项目立项、培训课件），由试点团队使用并反馈
2. **模板扩展**：根据企业品牌色和 VI 规范，新增企业专属风格到 style-system.md
3. **平台集成**：将 Skill 集成到企业内部 AI Agent 平台，降低使用门槛
4. **知识库对接**：将企业内部知识库作为 Step 2 素材源，产出更贴合业务的 PPT
5. **效果量化**：建立使用统计看板，追踪使用频次、节省工时、用户满意度

### 适用范围扩展

- **企业内部**：汇报材料、培训课件、项目提案、竞标文档
- **对外展示**：产品介绍、客户方案、路演 Deck
- **知识管理**：将长篇文档/报告自动转化为易于传播的演示格式
