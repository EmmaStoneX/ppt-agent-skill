# PPT Agent Skill 用户使用手册

> **一句话说明**：告诉 AI 你想做什么 PPT，它会像专业设计公司一样帮你从零生成一份高质量演示文稿。
>
> **期望管理**：生成的 PPT 能达到 **80-90% 完成度**（结构合理、风格统一、数据可视化到位），局部细节（文字溢出、间距微调）仍需 10 分钟人工润色。
>
> **预期耗时**：约 **25 分钟**（不配图）/ 45 分钟（配图）。

## 1. 使用前准备

### 必须项

| 依赖 | 最低版本 | 验证命令 | 说明 |
|------|---------|---------|------|
| Python | >= 3.8 | `python --version` | 运行后处理脚本 |
| Node.js | >= 18 | `node --version` | HTML 转 SVG 转换引擎 |
| pip 依赖 | — | `pip install python-pptx lxml Pillow` | 生成 PPTX 文件所需 |

> **重要**：Puppeteer 和 dom-to-svg 首次安装需要下载 Chromium 浏览器（约 170MB），耗时较长。
> 如果不提前安装，生成 PPT 到最后一步（HTML → PPTX）时才会触发安装，**很可能因超时导致会话中断**。
>
> 请**逐行执行**以下命令（不要一次性粘贴）：
> ```bash
> mkdir -p ~/.openclaw/workspace/skills/ppt-agent/ppt-output
> cd ~/.openclaw/workspace/skills/ppt-agent/ppt-output
> npm init -y
> npm install puppeteer dom-to-svg
> ```
> `ppt-output` 是技能的固定输出目录，所有 PPT 项目都会生成在它的子文件夹中。
> 在这里安装一次依赖，后续所有项目都能共用，无需重复安装。

### 可选项

| 配置 | 用途 | 不配会怎样 |
|------|------|-----------|
| `BRAVE_API_KEY` | 网页搜索（免费 2000 次/月） | AI 用自身知识替代，内容可能不够新 |
| `TAVILY_API_KEY` | 网页搜索 + 内容提取 | 同上 |
| `IMAGE_API_KEY` | AI 配图（Gemini 生图） | 不配图，用纯 CSS 装饰替代（推荐） |

**配置方法**：
```bash
cp .env.example .env
# 用文本编辑器打开 .env，填入你的 API Key
```

## 2. 30 秒快速开始

**最简触发方式** — 直接对 AI 说：

```
帮我做一个关于 XX 的 PPT
```

就这一句话。接下来：

1. AI 会问你 **7 个问题**（了解你的场景、受众、需求）
2. 你回答完 → AI 自动完成剩余所有步骤（搜索资料 → 大纲 → 策划 → 设计 → 输出 PPTX）
3. **全程你只需要回答一次问题，其他都是自动的**

**其他触发方式**：

| 你说的话 | 效果 |
|---------|------|
| "做一个关于 X 的 PPT" | 标准流程 |
| "做 15 页暗黑风的 AI 安全汇报材料" | 跳过部分已知问题 |
| "把这篇文档做成 PPT" | 以你的文档为素材 |
| "我要给老板汇报 Y" | 自动识别为 PPT 需求 |
| "做个培训课件" / "做路演 deck" | 同样触发 |

## 3. 完整工作流指南

### Step 1：需求调研（~3 分钟）

| | |
|---|---|
| **你需要做什么** | 回答 AI 提出的 7 个问题 |
| **系统在做什么** | 先搜索你的主题背景，再根据搜索结果定制问卷 |
| **产出** | 需求定义（内部使用，你不需要操心） |

#### 7 个问题详解与选择建议

**第一层：场景与受众**

| # | 问题 | 影响什么 | 选择建议 |
|---|------|---------|---------|
| 1 | **演示场景** — 现场演讲 / 自阅文档 / 培训教学 | 文字密度和视觉风格 | 现场演讲 → 文字少图多有冲击力；自阅文档 → 信息完整能独立理解；培训 → 结构化知识点 |
| 2 | **核心受众** — AI 根据主题推断 3-4 种画像 | 专业深度和说服策略 | 选最符合的那个，不确定就选"其他"并描述 |
| 3 | **期望行动** — 决策 / 理解记住 / 掌握方法 / 改变认知 | 内容编排方向 | 汇报选"决策"，科普选"理解"，培训选"掌握方法" |

**第二层：内容策略**

| # | 问题 | 影响什么 | 选择建议 |
|---|------|---------|---------|
| 4 | **叙事结构** — 问题→方案→效果 / 是什么→为什么→怎么做 / 对比论证 / 时间线 | 大纲骨架 | 商务汇报选"问题→方案→效果"，科普选"是什么→为什么→怎么做" |
| 5 | **内容侧重** — 根据搜索结果动态生成，可多选 | 各章节权重 | 选 2-3 个你最想强调的方向 |
| 6 | **说服力要素** — 数据 / 案例 / 权威背书 / 流程方法，可多选 | 卡片内容类型 | 给老板看选"数据"，给客户看选"案例"，做培训选"流程方法" |

**第三层：执行细节**

| # | 问题 | 说明 |
|---|------|------|
| 7 | **补充信息**（自由填写） | 演讲人姓名、日期、品牌色、页数偏好、必须包含/回避的内容、是否配图 |

> **选择困难？** 大部分问题直接选 A 就够用了。留空的 AI 会自动决定。

---

### Step 2：资料搜集（~3 分钟）

| | |
|---|---|
| **你需要做什么** | 无需操作，等待即可 |
| **系统在做什么** | 用搜索引擎查找主题相关资料（需配置 API Key） |
| **产出** | 搜索结果（内部使用） |

> 没有配置搜索 API Key？AI 会用自身知识替代，结果可能不如实时搜索准确。

---

### Step 3：大纲策划（~2 分钟）

| | |
|---|---|
| **你需要做什么** | 无需操作 |
| **系统在做什么** | 基于金字塔原理（结论先行、逻辑递进）生成大纲结构 |
| **产出** | 大纲 JSON（内部使用） |

---

### Step 4：内容分配 + 策划稿（~5 分钟）

| | |
|---|---|
| **你需要做什么** | 无需操作，AI 自动完成 |
| **系统在做什么** | 为每一页分配内容、选择布局、设计卡片结构 |
| **产出** | 策划稿（AI 会展示概览供你参考，然后自动继续下一步） |

> AI 会展示策划稿概览让你了解整体结构。如果你觉得有问题，可以随时打断提出修改意见；如果没问题，AI 会自动继续。

---

### Step 5：风格设计 + 逐页生成（~10 分钟）

| | |
|---|---|
| **你需要做什么** | 无需操作，等待即可（这一步最耗时） |
| **系统在做什么** | 选择视觉风格 → 匹配图标 → 逐页手写 HTML 设计稿 |
| **产出** | 每页 HTML 文件 |

---

### Step 6：后处理 + 输出（~2 分钟）

| | |
|---|---|
| **你需要做什么** | 无需操作 |
| **系统在做什么** | HTML → 合并预览 → SVG → PPTX |
| **产出** | 最终交付文件（见[输出产物使用指南](#6-输出产物使用指南)） |

## 4. 风格选择指南

AI 会根据你的主题自动选择风格。你也可以在 Step 1 第 7 题指定。

### 6 种预置风格

| 风格 | 适用场景 | 视觉特点 | 背景 |
|------|---------|---------|------|
| **科技年终汇报** | 年终总结、技术成果、产品发布 | 深色 + 蓝橙科技感 | 深色 |
| **展会展览** | MWC/CES 展会、品牌展示、国际会议 | 纯黑 + 大气光效 | 深色 |
| **扁平插画汇报** | 月度/季度汇报、部门总结 | 白底 + 蓝色清爽 | 浅色 |
| **扁平插画培训** | 新员工培训、入职引导、内训课件 | 白底 + 活泼蓝色 | 浅色 |
| **蓝色科技互联网** | AI/互联网/SaaS 项目、技术方案 | 深蓝 + 青色霓虹 | 深色 |
| **蓝色立体活动策划** | 活动策划、营销推广、品牌活动 | 浅灰蓝 + 立体卡片 | 浅色 |

> **不知道选哪个？** 留空即可，AI 根据你的主题自动推断。默认使用"扁平插画汇报"（最通用）。

### 自定义品牌色

在 Step 1 第 7 题的补充信息中写：

```
品牌主色 #1DA1F2，背景用深色
```

AI 会基于最接近的预置风格，替换为你的品牌色。只需提供主色即可，其他配色自动计算。

## 5. 配图说明

### 默认不配图（推荐）

PPT 默认使用**纯 CSS 装饰**（渐变、几何图形、矢量图标），视觉效果已经很好，且：
- 生成速度快（~25 分钟完成）
- 100% 成功率，不依赖外部 API
- 文件体积小

### 开启配图

如果你想让 PPT 包含 AI 生成的插图，需要**同时满足**：

| 条件 | 怎么做 |
|------|-------|
| 在 `.env` 配置 `IMAGE_API_KEY` | 参见[使用前准备](#1-使用前准备) |
| 在 Step 1 第 7 题选择配图 | 选 B（关键页面配图）或 C（每页配图） |

**开启后的代价**：

| 项目 | 不配图 | 关键页面配图 | 每页配图 |
|------|-------|------------|---------|
| 总耗时 | ~25 分钟 | ~35 分钟 | ~45+ 分钟 |
| API 依赖 | 无 | 需要 IMAGE_API_KEY | 需要 IMAGE_API_KEY |
| 成功率 | 100% | ~50%（失败自动降级为 CSS 装饰） | ~50% |

> **建议**：首次使用先不配图，熟悉流程后再尝试。

## 6. 输出产物使用指南

所有文件输出到 `ppt-output/{主题}_{日期}/` 目录，例如 `ppt-output/AI安全培训_20260326/`。

### 最终交付文件

| 文件 | 怎么用 |
|------|-------|
| **`{主题}_{日期}.pptx`** | 用 WPS 或 Office 2016 打开即可查看和编辑 |
| **`{主题}_{日期}_preview.html`** | 双击用浏览器打开，可翻页预览最终效果（推荐先用这个检查） |
| **`svg/*.svg`** | 每页的矢量文件，也可单独拖入 PPT 编辑 |

### PPTX 编辑方法

1. 用 **WPS** 或 **Office 2016** 打开 `.pptx` 文件
2. 右键点击页面中的内容
3. 选择 **"转换为形状"**
4. 现在你可以自由编辑文字、移动元素、修改颜色了

> **注意**：如果你的 Office/WPS 版本不支持"转换为形状"，可以直接在 SVG 文件上操作，或用 `preview.html` 截图后重新排版。

### 人工润色建议

拿到 PPT 后，建议花 10 分钟检查：

| 检查项 | 常见问题 | 怎么修 |
|-------|---------|-------|
| 文字溢出 | 个别卡片文字超出边界 | 缩小字号或精简文字 |
| 间距微调 | 元素间距不够均匀 | 手动拖拽调整 |
| 数据核实 | AI 搜索的数据可能过时 | 替换为你掌握的最新数据 |
| 品牌一致 | 颜色/字体可能和公司规范有微小差异 | 统一替换 |

## 7. 常见问题 FAQ

### Q：没有 API Key 能用吗？
**A**：能用。搜索 API Key 和配图 API Key 都是可选的。没有搜索 Key，AI 用自身知识替代；没有配图 Key，用 CSS 装饰替代。核心的 PPT 生成功能不需要任何 API Key。

### Q：配图失败了怎么办？
**A**：不用担心。配图失败的页面会自动降级为纯 CSS 装饰（渐变/几何图形），不影响最终 PPT 输出。

### Q：Node.js 版本低于 18 怎么办？
**A**：HTML → SVG 的转换需要 Node.js >= 18。如果版本低，系统会自动降级为备用方案（文字变路径，不可编辑）。建议升级 Node.js：
```bash
# 使用 nvm 升级（推荐）
nvm install 18
nvm use 18
```

### Q：生成中途中断了怎么办？
**A**：中间产物（大纲、策划稿、已生成的 HTML 页面）都保存在输出目录中。重新开始对话，告诉 AI "继续上次的 PPT 生成，输出目录是 ppt-output/xxx/"，可以从中断处继续。

### Q：PPTX 打开后内容显示异常？
**A**：
1. 尝试用 **WPS 最新版** 或 **Office 2016** 打开
2. 右键 → "转换为形状" 后再编辑
3. 如果仍有问题，可以用 `preview.html` 在浏览器中预览正确效果，对照调整 PPTX

### Q：可以指定页数吗？
**A**：可以。在 Step 1 第 7 题的补充信息中写明，比如"15 页"。留空则 AI 根据内容量自动决定（通常 8-15 页）。

### Q：生成的 PPT 可以商用吗？
**A**：可以。项目基于 MIT 许可证，生成的 PPT 内容归你所有。但请注意核实 AI 搜索填入的数据准确性。

## 8. 场景案例

### 案例 1：年终汇报（标准流程）

**你说**：
```
帮我做一个 2025 年度技术部门年终汇报 PPT
```

**AI 问你 7 个问题，你回答**：
1. 演示场景 → A. 现场演讲
2. 核心受众 → 公司高管和技术负责人
3. 期望行动 → A. 做出决策（审批明年预算）
4. 叙事结构 → A. 问题→方案→效果
5. 内容侧重 → 技术成果 + 团队建设
6. 说服力要素 → A. 硬数据 + D. 流程方法
7. 补充信息 → 演讲人：张三，12 页左右，品牌色 #1DA1F2

**最终产出**：12 页科技年终汇报风格 PPT，包含数据可视化图表，~25 分钟完成。

**最终产出**：10 页展会展览风格 PPT，封面和章节页有 AI 生成配图，~35 分钟完成。
